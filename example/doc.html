Quick Start
Core structure used across most pages

The common Retro-term pattern is similar to a Bootstrap admin layout: rt-admin acts as the shell, rt-sbr is the sidebar, and rt-main contains the page content. 

<div class="rt-admin">
  <aside class="rt-sbr">...</aside>
  <main class="rt-main">
    <div class="rt-topbar">...</div>
    <div class="card rt-section">...</div>
  </main>
</div>
Why Retro-term
A lighter, dashboard-first alternative to Bootstrap-style setups

Retro-term is built for admin dashboards and product consoles where you want a smaller UI surface, fewer design decisions, and a consistent visual language out of the box.

Lighter footprint

The framework focuses on the components this project uses most, so the design system stays compact and easier to maintain.

Dashboard-first

Navigation, tables, forms, badges, modal, and toast patterns are tuned for admin workflows instead of a general-purpose site kit.

Less setup

You get a ready-made visual system with fewer moving parts, which reduces the amount of custom CSS needed in each page.

Class Names
Naming rules used in this framework
Pattern Meaning
rt-* Main namespace for framework classes.
*.is-active / *.is-open Active or open state for interactive components.
*--variant Modifier for component color, size, or mode.
component__part Inner component parts such as icon, content, or footer.
Layout
Shell, grid, flow, section, and spacing
  • rt-admin: main wrapper for sidebar + content.
  • rt-main: primary content area to the right of the sidebar.
  • rt-grid: general grid for cards/content.
  • rt-flow: flex wrap with consistent gaps.
  • rt-section: section container with padding and spacing.

Overlays
Modal, toast, close button, backdrop

Overlays are used for important interactions such as confirmations, short forms, and temporary notifications. State classes like is-open and is-closing control the animations.

Class Function
modal Fullscreen backdrop for dialogs.
modal-content Main dialog container.
modal-header Title area and close button.
modal-body Dialog body content.
modal-footer Action area at the bottom.
rt-toast Transient notification.
Utilities
Spacing, flow, row, helper, and state classes
  • rt-flow and its modifiers control spacing between items.
  • rt-row is used for flex-wrap based layouts.
  • rt-mt1 through rt-mt4 provide top margin.
  • rt-p1 through rt-p4 provide padding.
  • rt-row-cols-* is used for row-based responsive grids.
